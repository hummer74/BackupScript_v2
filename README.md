# BackupScript_v2

Набор PowerShell-скриптов для автоматического резервного копирования виртуальных машин Hyper-V с трёхуровневой стратегией ротации архивов (GFS — Grandfather-Father-Son) и уведомлениями об ошибках по электронной почте.

## Обзор решения

Скрипты предназначены для ежедневного автоматического бэкапа виртуальных машин Hyper-V на сервере под управлением Windows Server. Решение обеспечивает:

- **Автоматическое резервное копирование** — виртуальная машина корректно останавливается, экспортируется, сжимается в 7z-архив и запускается обратно. Простой минимизирован за счёт того, что ВМ перезапускается сразу после экспорта, а сжатие происходит параллельно.
- **Трёхуровневую ротацию GFS** — ежедневные копии хранятся 5 дней, еженедельные — 4 недели, ежемесячные — 6 месяцев. Это позволяет при необходимости откатиться на любую дату в пределах полугодия.
- **Проверку целостности архивов** — после сжатия каждый архив проходит проверку встроенной командой `7z t`.
- **Контроль свободного места** — перед запуском скрипт проверяет, достаточно ли свободного пространства на диске для экспорта ВМ и итогового архива.
- **Уведомления об ошибках** — при возникновении любых ошибок формируется и отправляется email-уведомление через SMTP (поддержка Yandex и других провайдеров).
- **Детальное журналирование** — все действия записываются в общий лог-файл с метками времени, типом сообщения и названием источника.

## Архитектура ротации

Решение реализует классическую стратегию **Grandfather-Father-Son (GFS)** с тремя уровнями хранения:

```
F:\HyperV-B\
├── 00_Daily\       ← Ежедневные бэкапы (хранятся 5 дней)
│   ├── 1c-azimut-2026-04-20_20-30.7z
│   ├── 1c-azimut-2026-04-21_20-30.7z
│   └── ...
├── 01_Weekly\      ← Еженедельные копии (хранятся 4 недели)
│   ├── 1c-azimut-2026-04-18_20-30.7z
│   └── ...
└── 02_Monthly\     ← Ежемесячные копии (хранятся 6 месяцев)
    ├── 1c-azimut-2026-03-21_20-30.7z
    └── ...
```

**Цепочка ротации работает следующим образом:**

1. **Backup-HyperV.ps1** (ежедневно, пн–пт) — создаёт бэкап ВМ и помещает архив в `00_Daily`. Автоматически удаляет архивы старше 5 дней.
2. **Rotate-Weekly.ps1** (еженедельно) — копирует последний ежедневный архив в `01_Weekly`. Удаляет еженедельные архивы, превышающие лимит в 4 штуки.
3. **Rotate-Monthly.ps1** (ежемесячно) — копирует последний еженедельный архив в `02_Monthly`. Удаляет ежемесячные архивы, превышающие лимит в 6 штук.

BackupScript_v2/

## Подготовка к работе

### Предварительные требования

| Компонент | Требование |
|-----------|-----------|
| ОС | Windows Server 2012 R2 / 2016 / 2019 / 2022 |
| Hyper-V | Модуль Hyper-V установлен и роль включена |
| PowerShell | Версия 5.1 или выше |
| 7-Zip | Установлен (по умолчанию ожидается в `C:\Program Files\7-Zip\7z.exe`) |
| Права | Запуск от имени пользователя с правами администратора Hyper-V |
| SMTP | Доступ к SMTP-серверу для отправки уведомлений |

### Настройка конфигурации

Перед первым запуском откройте `Backup-HyperV.ps1` и отредактируйте блок конфигурации в начале файла:

```powershell
# === CONFIGURATION ===
$VMName          = "1c-azimut"          # Имя вашей виртуальной машины
$BackupRoot      = "F:\HyperV-B"        # Корневой каталог бэкапов
$DailyPath       = Join-Path $BackupRoot "00_Daily"
$TempRoot        = "E:\Z-TEMP\HyperV-Backup"  # Временная папка для экспорта
$SevenZipPath    = "C:\Program Files\7-Zip\7z.exe"  # Путь к 7z.exe
$LogFile         = "E:\HyperV-Backup.log"          # Общий файл журнала
$DaysToKeep      = 5                    # Сколько ежедневных архивов хранить
```

Аналогично отредактируйте конфигурацию в `Rotate-Weekly.ps1` и `Rotate-Monthly.ps1`, если нужно изменить пути или количество хранимых архивов.

### Настройка почтовых уведомлений

Создайте файл `C:\Script\MailConfig.ps1` со следующими переменными (замените значения на свои):

```powershell
# === SMTP Configuration ===
$SmtpServer  = "smtp.yandex.ru"
$SmtpPort    = 587
$EnableSsl   = $true
$SmtpUser    = "your-login@yandex.ru"
$SmtpPassword = "your-app-password"       # Используйте пароль приложения, не основной!
$MailFrom    = "your-login@yandex.ru"
$MailTo      = "admin@your-domain.ru"     # Адрес получателя уведомлений
```

> **Совет:** Для Yandex.Mail используйте [пароль приложения](https://yandex.ru/support/mail/id/app-passwords.html), а не основной пароль от аккаунта. Для других провайдеров укажите соответствующий SMTP-сервер и порт.

## Развертывание

### Размещение файлов

1. Скопируйте все скрипты из репозитория в рабочую директорию, например `C:\Script\`:

```
C:\Script\
├── MailConfig.ps1       # Создать вручную (см. выше)
├── MailFunctions.ps1
├── Backup-HyperV.ps1
├── Rotate-Weekly.ps1
└── Rotate-Monthly.ps1
```

2. Убедитесь, что директории бэкапов (`F:\HyperV-B\00_Daily`, `01_Weekly`, `02_Monthly`) и временная папка (`E:\Z-TEMP\HyperV-Backup`) существуют или могут быть созданы скриптом автоматически.

### Регистрация задач в Планировщике

В репозитории включены XML-файлы с готовыми определениями задач для Планировщика задач Windows. Для импорта выполните от имени администратора:

```powershell
# Импорт задачи ежедневного бэкапа
schtasks /Create /TN "00 Hyper-V Backup" /XML "C:\Script\00 Hyper-V Backup.xml" /F

# Импорт задачи еженедельной ротации
schtasks /Create /TN "02 Hyper-V Rotate Weekly" /XML "C:\Script\02 Hyper-V Rotate Weekly.xml" /F

# Импорт задачи ежемесячной ротации
schtasks /Create /TN "03 Hyper-V Rotate Monthly" /XML "C:\Script\03 Hyper-V Rotate Monthly.xml" /F
```

> **Внимание:** Перед импортом откройте XML-файлы и при необходимости скорректируйте пути к скриптам, а также данные учётной записи в секции `<Principals>` (SID пользователя) под вашу среду.

## Описание скриптов

### Backup-HyperV.ps1

Основной скрипт, выполняющий полный цикл резервного копирования виртуальной машины. Запускается ежедневно по будням.

**Процесс выполнения:**

1. **Проверки перед запуском:**
   - Проверяет наличие 7-Zip по указанному пути.
   - Проверяет существование виртуальной машины с заданным именем.
   - Создаёт недостающие директории.
   - Проверяет свободное место на дисках (для временной папки и итогового архива), ориентируясь на суммарный размер VHD-дисков ВМ с 10-процентным запасом.

2. **Остановка виртуальной машины:**
   - Выполняется штатное завершение работы (graceful shutdown) с таймаутом 600 секунд.
   - Если ВМ не останавливается — принудительная остановка (`Turn Off`) с дополнительным таймаутом 60 секунд.
   - При неудаче обеих попыток бэкап прерывается, а временные файлы сохраняются для диагностики.

3. **Экспорт ВМ:**
   - Команда `Export-VM` выгружает полную конфигурацию и диски виртуальной машины во временную папку.

4. **Перезапуск ВМ:**
   - Виртуальная машина запускается сразу после завершения экспорта для минимизации простоя.
   - Ожидание перехода в состояние `Running` — до 300 секунд с интервалом проверки 15 секунд.
   - В блоке `finally` предусмотрена дополнительная проверка: если ВМ по какой-то причине не работает, скрипт попытается запустить её повторно.

5. **Сжатие:**
   - Временная папка с экспортом сжимается в 7z-архив с уровнем сжатия `mx5` (нормальный).
   - Имя архива формируется по шаблону: `ИмяВМ-YYYY-MM-DD_HH-MM.7z`.

6. **Проверка целостности:**
   - Запускается `7z t` для верификации созданного архива.
   - При ошибке проверки архив не удаляется — он остаётся для ручного анализа.
   - Только при успешной проверке временная папка экспорта удаляется.

7. **Ежедневная ротация:**
   - Скрипт сканирует директорию ежедневных архивов и удаляет файлы старше `$DaysToKeep` дней.
   - Дата извлекается из имени файла (формат `YYYY-MM-DD`). Если имя не соответствует шаблону, используется `LastWriteTime` файла.

8. **Уведомления:**
   - При любой ошибке отправляется email через `Send-ErrorNotification`.

### Rotate-Weekly.ps1

Скрипт еженедельной ротации. Копирует самый свежий ежедневный архив в папку еженедельных бэкапов и удаляет старые копии, превышающие лимит.

**Ключевые параметры:**
- `$WeeksToKeep = 4` — количество еженедельных архивов, которые необходимо хранить.

**Логика работы:**
1. Находит последний (по `LastWriteTime`) ежедневный архив в `00_Daily`.
2. Копирует его в `01_Weekly`, если копия с таким именем ещё не существует (защита от дублирования).
3. Сортирует еженедельные архивы по дате и удаляет самые старые, превышающие лимит `$WeeksToKeep`.
4. Удаление старых архивов пропускается, если копирование нового архива завершилось ошибкой.

---

### Rotate-Monthly.ps1

Скрипт ежемесячной ротации. Работает аналогично недельному, но берёт архив из `01_Weekly` и помещает его в `02_Monthly`.

**Ключевые параметры:**
- `$MonthsToKeep = 6` — количество ежемесячных архивов, которые необходимо хранить.

---

### MailFunctions.ps1

Вспомогательный модуль с единственной функцией `Send-ErrorNotification`, которая используется всеми остальными скриптами.

**Реализация:**
- Подключается к SMTP-серверу (поддержка SSL, порт 587).
- Использует аутентификацию по логину и паролю (переменные из `MailConfig.ps1`).
- Кодировка тела и темы письма — UTF-8 (поддержка кириллицы).
- Таймаут соединения — 10 секунд.

```powershell
# Пример вызова из любого скрипта:
Send-ErrorNotification -Subject "Hyper-V Backup ERROR: $VMName" -Body "Описание ошибки..."
```

## Настройка параметров ротации

Стратегия ротации легко адаптируется под ваши нужды путём изменения переменных в конфигурационных блоках скриптов:

| Параметр | Скрипт | По умолчанию | Описание |
|----------|--------|-------------|----------|
| `$DaysToKeep` | `Backup-HyperV.ps1` | `5` | Количество хранимых ежедневных архивов |
| `$WeeksToKeep` | `Rotate-Weekly.ps1` | `4` | Количество хранимых еженедельных архивов |
| `$MonthsToKeep` | `Rotate-Monthly.ps1` | `6` | Количество хранимых ежемесячных архивов |

Например, для увеличения глубины хранения достаточно изменить значения:

```powershell
# В Backup-HyperV.ps1 — хранить ежедневные архивы 7 дней
$DaysToKeep = 7

# В Rotate-Weekly.ps1 — хранить 8 недельных копий
$WeeksToKeep = 8

# В Rotate-Monthly.ps1 — хранить 12 ежемесячных копий (год)
$MonthsToKeep = 12
```

## Устранение неполадок

### Скрипт завершается с ошибкой «7-Zip not found»

Убедитесь, что 7-Zip установлен и путь в переменной `$SevenZipPath` указывает корректно. Проверить можно так:

```powershell
Test-Path "C:\Program Files\7-Zip\7z.exe"
```

### Виртуальная машина не останавливается

- Проверьте, что внутри гостевой ОС установлены и работают компоненты интеграции Hyper-V (Integration Services) — они обеспечивают корректное завершение работы.
- При проблемах с graceful shutdown скрипт автоматически применяет принудительную остановку.
- Убедитесь, что keine другие процессы или пользователи блокируют выключение ВМ.

### Не хватает свободного места на диске

Скрипт проверяет свободное пространство перед началом работы. Ошибка с описанием дефицита места будет записана в лог и отправлена по email. Возможные решения:

- Увеличьте объём диска или подключите дополнительный.
- Снизьте уровень сжатия (измените `mx5` на `mx3` или `mx1` в команде 7-Zip).
- Уменьшите `$DaysToKeep` для более агрессивной очистки старых архивов.

### Email-уведомления не приходят

- Проверьте корректность данных в `MailConfig.ps1`.
- Для Yandex.Mail убедитесь, что используется пароль приложения, а не основной пароль аккаунта.
- Убедитесь, что SMTP-порт (587) не заблокирован firewall.
- Проверьте лог скрипта — там будет указана конкретная ошибка SMTP.

### Временная папка не очищается

Это нормальное поведение при возникновении ошибок. Скрипт намеренно оставляет временную папку `$TempExportFolder` для диагностики. Удалите её вручную после разбора проблемы:

```powershell
Remove-Item -Path "E:\Z-TEMP\HyperV-Backup" -Recurse -Force
```

## Безопасность

- **Учётные данные SMTP** хранятся в файле `MailConfig.ps1`, который исключён из репозитория через `.gitignore`. Не публикуйте этот файл в открытых системах контроля версий.
- **Права выполнения** — скрипты требуют прав администратора Hyper-V для управления виртуальными машинами. Рекомендуется запускать их от имени выделенной сервисной учётной записи.
- **Планировщик задач** — XML-файлы содержат SID пользователя. При переносе на другой сервер обновите секцию `<Principals>` на актуального пользователя.
- **Журнал** — общий лог-файл (`E:\HyperV-Backup.log`) содержит подробную информацию о всех операциях. Регулярно проверяйте его или настройте ротацию лог-файла.
